# AI-N8N

🚀 AI 內容生產線：基於 n8n 的自動化多代理系統 (MAS)

這是一個利用 n8n 流程自動化工具，結合 Google Gemini API 所建構的「自動化內容生成與交付系統」。本專案模擬了專業編輯部的協作流程，實現了從「一鍵選題」到「精美網頁信件發送」及「雲端備份」的完整自動化流程。

🌟 核心亮點 (Project Highlights)

多代理系統 (Multi-Agent System, MAS) 架構：

Planner (內容規劃師)：負責結構分析，產出邏輯嚴謹的文章大綱。

Writer (專業作家)：根據大綱擴寫內容，確保專業的繁體中文語氣。

Editor (社群媒體編輯)：負責將文字轉化為具備 CSS 樣式的精美 HTML/CSS 日報格式。

技術突破：物理級內容過濾器 (V7 Logic)：

痛點：解決 AI 預覽版模型（如 Gemini 2.5-flash）容易出現「重複指令 (Parrot-speak)」或夾雜廢話的現象。

方案：開發 JavaScript 核心過濾邏輯，利用 Regex (正則表達式) 與 最大區間擷取法，強制過濾 AI 廢話，確保輸出 100% 為純淨 HTML 代碼。

魯棒性設計 (Robustness)：

內建 If-Else 錯誤處理機制，當 AI 回傳空值時自動導向補救路徑（Wait & Retry），防止自動化生產線中斷。

端到端自動交付 (E2E Delivery)：

整合 Gmail API 自動寄送卡片式排版郵件。

整合 Google Drive API 實現雲端檔案自動備份（儲存為 .html 檔案）。

🛠️ 技術棧 (Tech Stack)

AI Engine: Google Gemini API (1.5-pro / 2.5-flash)

Logic Execution: JavaScript (Node.js Environment)

Cloud Integrations: Google Gmail API, Google Drive API

⚙️ 安裝與設定 (Setup)

1. 匯入工作流

下載本倉庫的 workflow.json 並在 n8n 介面選擇「Import from File」。

2. 設定 API 憑證

Gemini API: 前往 Google AI Studio 申請 API Key。

Google OAuth2:

前往 Google Cloud Console 啟用 Gmail 與 Drive API。

設定 OAuth Client ID 與 Secret。

在 n8n 中完成授權 (Redirect URI 需與 n8n 提供的網址一致)。
